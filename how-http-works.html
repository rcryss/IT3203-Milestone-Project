How HTTP Works

HTTP was invented in 1989 by Tim Berners-Lee, while he was working at CERN. HTTP was designed to facilitate communication between scientists allowing them to share information at a faster rate. HTTP is an application layer protocol that allows web browsers and servers to communicate with each other across the Internet.

The Client-Server Model

HTTP is based on the client-server model. A client, such as a web browser, sends an HTTP request to a server, in order to access a resource, such as a webpage or another online resource. A common request method is GET, which is used to retrieve information from a server. After recieving the request, the server then processes it and returns a response which may be a status code or the requested webpage or resource. Another request method is POST, which is when data is sent from the client to the server, such as when logging onto a website, filling out a form, or making a purchase.


HTTP Responses

After a web server recieves and processes an HTTP request, it then returns a response to the client. The response consists of a status code, headers, and a body. The status code identifies whether the request was sucessful or if there was an error. Response headers provide additional information about the response, such as the content type and length, server details and other information. The message body contains content that is requested by the client which may be a webpage, images, videos or other resources.

HTTP Status Codes

An HTTP status code is a three-digit response to a client's request. There are five different categories:

  • 1xx(Informational)
  • 2xx(Success)
  • 3xx(Redirection)
  • 4xx(Client Error)
  • 5xx(Server Error)
